Group Projects:

 

Topics

·         Develop an algorithm to reach consensus (or, k-value consensus) in mobile social networks

·         Develop an algorithm for task scheduling in everyday life

·         Develop an algorithm to find the minimum cost/energy path (shortest path?) between two points to be covered by your skateboard (while coming to school/class)

·         Develop an algorithm to make sure you wait for the minimum time while waiting for a bus connection

·         Develop an algorithm to update links in random/dynamic (social) graphs

·         Develop a distributed algorithm for a new event incorporation in an already existing schedule depending on certain constraints (like how to accommodate a new ride-sharing request in a taxi which already has certain pre-confirmed requests).

and many others ........

 

Group Project Policy

  • Students themselves should form the project groups and select a topic and then submit that information to the Instructor.
  • Each group should have 2-4 members. Single student projects are also possible but generally discouraged. Groups can have members across sections (A and B).
  • Each group can pick up one topic and submit (to the instructor) an A4 sheet (or by email) with names and MU-IDs of group members, title of the project chosen, and a brief description of the objective (what you want to do). GET INSTRUCTOR'S PERMISSION for your chosen topic.
  • The section of majority of the project group members will determine during which section they will have their presentation (e.g., if a project group has more than half members from 464 A, then it will be considered for presentation during the class time of 464 A). However, it will be decided later whether the presentation will be during the class or separately.
  • Project groups will have chances (strongly encouraged) to discuss their problem/progress during office hours of the Instructor or the TA.
